why we should use Redux in React

2018-01-17 20:46:45 | React&Redux

Why We Should Use

The Idea of developing everythings in Component released the hard working on operation on DOM. For React, everything is state, through the state can control the view changes. But as the project keeping expends and mroe and more function are intergrated in, many states become so unmanageable that as different components trigger changes in the same state and lead to updates to the view, we may not know exactly what happened and state changes have become somewhat unpredictable and not Controlled, Redux came into being, limiting the timing and manner of stateful updates through the practice and enhancement of Flux thinking, and Redux attempted to make state changes predictable.

Three Principles of Redux

1. Single data source.

The applications only have the single data source, the entire state of the application are stored in an object, which providing the persistence of extracting the status for entire application.Mean while, combineReducers (Redux API) provides a efficiently solution for dealing with large data sources

2.Read only State

In Redux, we change the state through reducer instead of setState() and depending on the type of action which being triggered . reducer(previousState, action) => newState

3.Using Pure function to complete the change of state

Using reducer to complete the state change, every reducer is a pure function
